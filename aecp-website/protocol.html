How AECP Works

Understanding the Agent Embedding Communication Protocol

1

Handshake & Capability Exchange

Agents introduce themselves and exchange embedding model information:

  • Model name and version
  • Embedding dimensions
  • Quality thresholds
  • Protocol version
{
  "agent_id": "agent_a",
  "embedding_model": "text-embedding-3-small",
  "dimensions": 1536,
  "min_quality_threshold": 0.75
}
2

Calibration

Agents learn to translate between embedding spaces using shared vocabulary:

Training/Validation Split

  • 80% Training: Used to compute transfer matrices
  • 10% Validation: Held-out for quality checking
  • 10% Test: Completely unseen data

Transfer Matrix Computation

Uses least squares regression to find optimal linear transformation:

WAB = argmin ||EA · W - EB||²

Where EA and EB are embedding matrices from agents A and B.

3

Semantic Transfer

Once calibrated, embeddings can be transferred instantly:

Quality Metrics

  • Forward Similarity: How well A→B transfer preserves semantics
  • Round-trip Similarity: How well A→B→A preserves original
  • Worst-case: Minimum similarity observed
4

Monitoring & Recalibration

Continuous quality monitoring ensures reliable communication:

  • Automatic Expiration: Matrices expire after 7 days (configurable)
  • Quality Threshold: Alert if similarity drops below 75%
  • Transfer Logging: Track all transfers for debugging
  • Recalibration: Re-compute matrices when needed

Technical Deep Dive

Matrix Computation

AECP uses numpy's least squares solver (lstsq) which employs LAPACK's DGELSD:

  • Computes SVD for numerical stability
  • Handles over-determined systems (n_samples > dimensions)
  • Regularization available for ill-conditioned matrices
  • O(n · d²) complexity where n=samples, d=dimensions

Quality Guarantees

Validated on 300,000 vocabulary items with zero overfitting:

  • Training similarity: 95.86%
  • Validation similarity: 97.34%
  • Test (unseen) similarity: 97.35%
  • Worst-case: 82.43%

Key Finding: No degradation on unseen data (0.01% difference from validation)
