How AECP Works

Linear mapping between embedding spaces — from calibration to serve.

1. Calibration

Embed ~2K texts with both the source and target models. This creates paired matrices X (source) and Y (target).

# Embed K calibration texts with both models
X = source_model.embed(calibration_texts)  # (K, d_src)
Y = target_model.embed(calibration_texts)  # (K, d_tgt)

Calibration texts should come from your corpus domain. Out-of-domain degrades quality.

2. Fit Mapping

Ridge regression finds the optimal linear transformation:

W = argmin ||X · W - Y||² + α||W||²

α is auto-tuned via leave-one-out GCV. Handles both same-dimension and rectangular pairs.

from aecp import RidgeMapping

m = RidgeMapping(alpha="auto", seed=0)
m.fit(X, Y)
m.save("mapping.aecp")

The .aecp file stores: forward W, inverse W_inv (independent α), optional recalibrator, metadata.

3. Serve (query-time mapping)

No corpus writes needed. Map new-model queries into legacy space on-the-fly:

New Modeld_tgt dim
→W-1
Legacy Spaced_src dim
from aecp.serve import QueryAdapter

qa = QueryAdapter.load("mapping.aecp")
legacy_vec = qa.map_query(new_model_embed(query))
results = legacy_index.search(legacy_vec, k=10)

4. Offline Migration (optional)

Transform stored corpus vectors. Source collection is never modified.

→W
New Model Spaced_tgt dim
from aecp.adapters.chroma import migrate_collection

report = migrate_collection(client, "docs", mapping)
print(f"Migrated {report.rows_processed} rows")

Math

  • Forward: Y ≈ X · W (ridge regression with bias column)
  • Inverse: X ≈ Y · W_inv (separate α, independently fitted)
  • Normalization: All outputs are L2-normalized
  • Complexity: O(K · d²) fit, O(N · d · d') transform
