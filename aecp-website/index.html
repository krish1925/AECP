import { AECP } from '@aecp/core';
import { OpenAIAdapter } from '@aecp/adapters-openai';
import { VoyageAdapter } from '@aecp/adapters-voyage';

// Initialize agents with different models
const agent1 = new AECP({ embedder: new OpenAIAdapter({ apiKey: "sk-..." }) });
const agent2 = new AECP({ embedder: new VoyageAdapter({ apiKey: "pa-..." }) });

// One-time calibration
await agent1.calibrateWith(agent2);

// Transfer embeddings between agents
const transfer = await agent1.transferTo(agent2.agentId, "machine learning");
// 97% semantic fidelity preserved! ✨

The Problem

Text Serialization Loses Information

When agents communicate through text, they lose 95% of semantic information stored in embeddings.

Incompatible Embedding Spaces

Different models create embeddings in different vector spaces. A 384d OpenAI embedding can't be used with a 768d Voyage model.

High Latency & Cost

Re-embedding text with each agent costs time and money. Text → Embed → Transfer → Re-embed adds 150ms+ latency.

The Solution: AECP

Direct Embedding Space Transfer

AECP learns linear transformation matrices that map embeddings between different models' vector spaces.

  • One-time calibration using shared vocabulary
  • Sub-millisecond transfer latency
  • 97% semantic fidelity on unseen data
  • Works with any embedding model
  • Zero overfitting (validated on 300k items)
