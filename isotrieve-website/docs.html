Isotrieve Documentation

Migrate a vector database to a new embedding model without re-embedding. Fits a linear map from ~2K calibration texts; 87–91% retrieval retention on BEIR.

What is Isotrieve?

Different embedding models produce vectors in incompatible geometric spaces. Raw cross-space nDCG@10 = 0.0. Isotrieve fits a ridge regression mapping from paired calibration texts, then transforms stored vectors (offline) or maps queries on-the-fly (serve mode).

Key Features

  • 87–91% retrieval retention on BEIR SciFact
  • Score recalibration — isotonic regression for reliable thresholds
  • Confidence flags — per-query high/medium/low
  • ChromaDB + LangChain adapters
  • Quality gate — data-driven PASS/WARN/FAIL

Installation

# Core only
pip install isotrieve

# With adapters
pip install "isotrieve[chroma]"       # ChromaDB
pip install "isotrieve[langchain]"    # LangChain
pip install "isotrieve[qdrant]"       # Qdrant
pip install "isotrieve[all]"          # Everything

Python ≥ 3.10, numpy ≥ 1.24, scikit-learn ≥ 1.3.

python -c "import isotrieve; print(isotrieve.__version__)"
# 0.2.0

Quick Start

1. Fit a mapping

import numpy as np
from isotrieve import RidgeMapping

rng = np.random.default_rng(0)
K, d_src, d_tgt = 500, 32, 48
X = rng.normal(size=(K, d_src))
Y = X @ rng.normal(size=(d_src, d_tgt))

m = RidgeMapping(alpha="auto", seed=0)
m.fit(X, Y)
m.save("mapping.isotrieve")

Z = m.transform(X[:10])  # (10, d_tgt), L2-normalized

2. Serve mode (zero corpus writes)

from isotrieve.serve import QueryAdapter

qa = QueryAdapter.load("mapping.isotrieve")
legacy_vec = qa.map_query(new_model_embed(query))

3. Offline migration

from isotrieve.migrate import migrate_store

migrate_store(
    source_dir="./old_store",
    target_dir="./new_store",
    mapping=m,
)

CLI

# Plan migration
isotrieve plan --source-model ada-002 --target-model te3-large --corpus-size 1000000

# Fit mapping
isotrieve calibrate --source-vectors X.npy --target-vectors Y.npy -o map.isotrieve

# Transform corpus
isotrieve transform --mapping map.isotrieve --source-dir ./old --target-dir ./new

# Inspect
isotrieve inspect map.isotrieve

Mapping Classes

RidgeMapping (default)

from isotrieve import RidgeMapping

m = RidgeMapping(alpha="auto", seed=0)
m.fit(X_cal, Y_cal)
m.save("mapping.isotrieve")

Z = m.transform(corpus)           # (N, d_tgt)
X_hat = m.inverse_transform(Z)    # back to source
  • Handles rectangular mappings (d_src != d_tgt)
  • Bias column for affine term
  • Independent inverse alpha (fitted separately)

OrthogonalProcrustesMapping

from isotrieve.mapping.linear import OrthogonalProcrustesMapping

m = OrthogonalProcrustesMapping(seed=0)
m.fit(X_cal, Y_cal)  # Requires d_src == d_tgt

Serve Mode

from isotrieve.serve import QueryAdapter

qa = QueryAdapter.load("mapping.isotrieve")
legacy_vec = qa.map_query(new_model_embed(query_text))
legacy_vecs = qa.map_queries(new_model_embed_batch(query_texts))

print(qa.d_src, qa.d_tgt)          # dimensions
print(qa.has_recalibrator)          # True if fitted

Offline Migration

from isotrieve.adapters.chroma import migrate_collection

report = migrate_collection(
    client, "docs", mapping,
    new_collection="docs_v2",
    batch_size=1000,
)
print(f"Migrated {report.rows_processed} rows")

When NOT to Use

  • Maximum quality matters more than cost
  • Calibration domain mismatches corpus
  • Quality gate returns FAIL
  • K < 2000

Anti-patterns

  • Don't mix vectors from different models in one collection
  • Don't assume same dimensionality = compatibility
  • Don't skip the quality gate
  • Don't use MLP (0.719 vs 0.866 for Ridge)

Security

  • Inversion attacks: Treat mapped vectors with same sensitivity as source text
  • API keys: Environment variables, never in code
  • Calibration: Use non-sensitive texts
