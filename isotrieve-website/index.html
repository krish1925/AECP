The Problem

Incompatible Embedding Spaces

Different models produce vectors in different geometric spaces. Raw cross-space nDCG@10 = 0.0.

1024d from bge ≠ 1024d from e5

Re-embedding Is Expensive

Switching models means re-embedding every document. 1M docs = hundreds of dollars.

Cost scales linearly with corpus size

Model Deprecation Looms

OpenAI deprecated ada-002. New models are better, but migrating means rebuilding from scratch.

ada-002 → text-embedding-3: same story

How Isotrieve Solves This

Fits a ridge regression from ~2K paired calibration texts. The learned mapping transforms stored vectors, or maps new-model queries into the legacy space.

  • >87–91% retrieval retention — measured on BEIR SciFact, 3 seeds
  • >Score recalibration — isotonic regression makes threshold decisions reliable
  • >Confidence flags — per-query high/medium/low with adaptive margins
  • >ChromaDB + LangChain adapters — drop-in wrappers for popular vector DBs
  • >Quality gate — data-driven PASS/WARN/FAIL from a trained model
